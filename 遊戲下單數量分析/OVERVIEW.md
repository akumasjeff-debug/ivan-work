# 遊戲下單數量分析 — 專案總覽

## 這個專案在做什麼

實體遊戲商品（卡片 / 主機 / 遊戲片 / 周邊等）進貨前的**決策分析**。
每一批要下單的遊戲，分析它的**網路聲量**與可比對的**需求線索**，最後給出
「**建議下單量**」與理由、風險，產出一份**分析報告**。

- 產出形式：Markdown 分析報告（不是互動工具、不寫程式）。
- 使用時機：每次進貨前，把「要評估的遊戲清單」丟進來 → 依 SOP 產一份報告。

## 內容分類

| 區塊 | 說明 |
|------|------|
| 聲量調查 | 各平台的討論度 / 熱度指標，換算成聲量分數 |
| 需求推估 | 對照歷史可比款、系列前作、平台與時機，推估銷量區間 |
| 下單建議 | 保守 / 基準 / 積極 三檔下單量，含理由與風險 |
| 總表 | 這批所有遊戲的建議一覽 |

## 檔案

| 檔案 | 用途 |
|------|------|
| `OVERVIEW.md` | 本檔，專案總覽 |
| `HANDOVER.md` | 分析方法論（SOP）、聲量評分規則、報告資料結構 |
| `報告範本.md` | 每次分析報告的空白範本 |
| `reports/` | 各批次的實際分析報告（依日期 / 批次命名） |

## 方法論一句話

**聲量（訊號）→ 需求推估（對照歷史）→ 下單量（加上成本與庫存限制）**，
每一步都寫清楚假設與資料來源，讓結論可被檢驗、可被修正。
