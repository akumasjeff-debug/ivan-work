# 交接 / 分析方法論（SOP）

這份文件定義「每次分析要怎麼做」，讓每份報告的流程與品質一致。
方法參考通用資料分析流程 + 存貨決策理論（EOQ、報童模型），但**簡化成實務可用**。

---

## 一、輸入：每款遊戲需要的基本資料

分析前，盡量蒐集每款遊戲下列欄位（缺的就標「未知」，別假裝有）：

| 欄位 | 說明 | 是否必要 |
|------|------|----------|
| 品名 | 遊戲 / 商品名稱 | 必要 |
| 平台 | Switch / PS5 / PC 實體 / 卡片 / 周邊… | 必要 |
| 類型 | RPG / 動作 / 卡牌 / 收藏品… | 建議 |
| 系列 / IP | 是否有前作、IP 知名度 | 建議 |
| 發售日 | 影響時機與季節性 | 建議 |
| 定價 / 進價 | 影響資金與積壓風險 | 建議 |
| 供應商 MOQ / 箱入數 | 最小下單單位（例如一箱 12 入） | 建議 |
| 退貨政策 | 能否退貨 → 影響要多保守 | 建議 |
| 我方過往同類銷量 | 最強的需求依據（若有） | 若有必附 |

> 決策限制（整批共用）：**總進貨預算上限、倉儲 / 貨架空間、資金週轉考量**。

---

## 二、步驟 1 — 網路聲量調查

對每款遊戲，蒐集下列訊號，每項給 0–5 分並註明來源與觀察值：

| 訊號來源 | 看什麼 |
|----------|--------|
| Google Trends | 關鍵字近 3–12 個月趨勢、是否上升 |
| 社群討論（巴哈姆特 / PTT / Dcard / Reddit） | 討論串數、回文熱度、正負評 |
| YouTube / 實況（Twitch） | 預告觀看數、實況主是否在玩、觀看人數 |
| Steam（若有 PC 版） | 願望清單排名、評論數與好評率 |
| 預購 / 銷售榜 | Amazon / 各通路預購排名 |
| 官方社群 | 粉專 / 官推互動數、追蹤成長 |
| 媒體報導 | 是否有大型媒體 / KOL 報導 |
| 在地化 | 是否中文化（對台灣市場影響大） |

**聲量分數**：把各項加權平均 → 換算成聲量等級。

| 等級 | 分數 | 意義 |
|------|------|------|
| 🔥 高 | 4.0–5.0 | 討論熱、趨勢上升、明確搶手 |
| ➖ 中 | 2.5–3.9 | 有一定關注，但不爆 |
| 🧊 低 | 0–2.4 | 冷門 / 小眾 / 訊號稀少 |

> 資料以「當下查到的」為準，一定要標日期與來源連結；聲量會隨時間變。

---

## 三、步驟 2 — 需求推估

聲量是「關注度」，不等於「會買多少」。要對照能換算成實際購買的線索：

1. **歷史可比款（analog）**：找系列前作 / 同類型 / 同平台、我方賣過的最接近品項，
   用它的實際銷量當基準。**這是最可靠的依據。**
2. **調整因子**（相對可比款，往上或往下修）：
   - 系列 / IP 熱度變化、平台裝機量
   - 是否中文化、是否獨佔
   - 發售時機 / 季節性（連假、年末、寒暑假）
   - 定價帶（越貴越保守）
   - 聲量等級（步驟 1 的結果）
3. 產出**需求區間**（低 / 中 / 高三個情境），而不是單一數字。

> 沒有歷史可比款時，明確說「純聲量推估、信心低」，並把區間拉寬。

---

## 四、步驟 3 — 下單量建議

用需求區間 + 成本 / 庫存限制，換算成實際下單量。核心觀念（報童模型）：
**缺貨的損失 vs 積壓的損失，哪個比較痛，就往哪邊偏。**

- 能退貨 / 熱門搶手 → 可積極（缺貨損失大於積壓）。
- 不能退貨 / 資金緊 / 冷門 → 要保守（積壓風險高）。

給**三檔建議**，並用 MOQ / 箱入數取整：

| 檔位 | 對應 | 適用 |
|------|------|------|
| 保守 | 需求區間下緣 | 資金緊、不可退、聲量低、信心低 |
| 基準 | 需求區間中位 | 一般情況的推薦值 |
| 積極 | 需求區間上緣 + 安全庫存 | 熱門搶手、可退貨、看好 |

每款都要寫：**建議下單量、理由、主要風險、信心程度（高/中/低）**。

---

## 五、步驟 4 — 產出報告

- 用 `報告範本.md` 複製一份，放到 `reports/`，檔名含日期與批次，
  例：`reports/2026-07_夏季進貨.md`。
- 每款一段（聲量摘要 → 需求推估 → 下單建議），最後一張**總表**彙整。
- 開頭寫**整批共用假設**（預算、日期、資料查詢時間）。
- 結尾寫**待確認事項 / 資料缺口**（哪些是猜的、需要你補資料的地方）。

---

## 六、原則（重要）

- **來源透明**：每個數字都要能追到來源與查詢日期；查不到就寫「未知」，不要編。
- **給區間不給假精準**：需求本質是估計，用區間 + 信心程度表達不確定性。
- **結論可執行**：最後一定落到「建議下單 X 個、因為 Y、風險 Z」。
- **可被推翻**：若你有內部銷售數字，一定會覆蓋純聲量的推估。
