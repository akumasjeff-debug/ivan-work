# 電玩庫存表 — 交接細節

## 架構

- 資料本體＝Google 試算表（網址見 `OVERVIEW.md`）；本 repo 放轉換腳本與建置腳本，無實際資料。
- 資料流：PChome 後台匯出檔（Downloads）→ `匯入轉換.py` → **POST 到試算表 Apps Script Web App（doPost）直接覆寫「庫存表」**；未設定 Web App 時退回剪貼簿手貼模式。
- Web App 驗證：token 存兩處——GAS 指令碼屬性 `API_TOKEN`（雲端）與本機 `config.local.json`（gitignore），兩邊要一致。網址與 token 都不進 git。
- `匯入轉換.py` 用 urllib POST；GAS Web App 回 302 轉址屬正常，urllib 會自動跟轉拿到 JSON 結果 `{ok, written, updatedAt}`。
- **改了 setup.gs 的程式碼後，Web App 要「部署 → 管理部署作業 → 編輯 → 新版本」才會生效**（GAS 部署是快照，這步最常忘）。
- v1 曾設計四分頁進出貨流水帳（商品主檔/進貨/出貨/總覽），2026-07-20 使用者提供 PChome 匯出檔後改為六欄匯入制，v1 設計已廢棄（git 歷史可查）。

## 匯出檔格式（匯入轉換.py 的輸入）

- 檔名 `20########_XXXX.xls`（時間戳開頭），**副檔名 .xls 但內容是 xlsx**——openpyxl 認副檔名會拒開，腳本先複製改名 .xlsx 再解析。
- 第一個工作表，第 1 列是欄名，共 14 欄。腳本**用欄名對欄**（不是用位置），取：
  `廠商名稱`、`商品編號(20碼含規格碼)`、`商品名稱`、`可賣量`、`成本`、`售價(網路價)`。
  PChome 改欄名時腳本會報「欄位對不上」並列出實際欄位。
- 沒商品編號的列跳過。售價 0（隱藏區組合用商品）與成本 0 屬正常資料，照實輸出。

## 匯入轉換.py 要點

- 不帶參數＝抓 Downloads 最新 `20########_*.xls`；可帶檔案路徑參數。
- 輸出 `reports/庫存表_YYYYMMDD.tsv`（utf-8-sig，Excel 可直開），同日重跑會覆蓋。
- 剪貼簿**不能用 clip.exe**（會把 UTF-16 BOM 留在內容裡，A1 帶隱形字元），改呼叫 PowerShell `Set-Clipboard` 讀 TSV 檔。
- 貼上是整份覆蓋（含標題列）；若新匯出比舊資料列數少，舊的多餘列會殘留——目前商品數只增不減所以沒事，要嚴謹就貼之前先清空資料列。

## setup.gs（Apps Script）要點

- 「庫存表」分頁：七欄標題（末欄來源廠商）、凍結首列、商品編號欄文字格式（防 20 碼被當數字）、數字格式、缺貨標紅、標題篩選器。
- 缺貨標紅＝條件格式 `=AND($B2<>"",$D2=0)` 套 `A2:G2000`；格式只套到 2000 列（`MAX_ROWS`），不夠調大重跑。
- 「可賣量歷史」分頁：A=商品編號（對列的鍵）、B=商品名稱、C 起一欄一日期。`updateHistory_` 邏輯：
  - 日期標籤由本機傳入（`匯入轉換.py` 取匯出檔檔名前 8 碼日期，對不上用當天）。
  - 標題列已有同日期 → 覆蓋同一欄（同一天重跑不會多欄）；否則最右加新欄。
  - 商品編號沒見過 → 補新列；這次匯出沒有的商品該日期欄留白（可看出下架）。
  - 商品名稱只在第一次補列時寫入，之後改名不回寫。
- 重跑 `setup` 不清資料列；篩選器會移除重建（`createFilter` 不能疊加）。

## 注意事項

- 成本屬敏感資料，**試算表不要開公開連結**；repo 內只有腳本，上傳 GitHub 無虞。
- `reports/` 內的 TSV 含成本，已被 `.gitignore` 排除（見根目錄 `.gitignore`），僅存本機。
