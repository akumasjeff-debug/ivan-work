# 電玩庫存表 — 交接細節

## 架構

- 資料本體＝使用者 Google 帳號下的一份試算表；本 repo 只有 `apps-script/setup.gs`（建置腳本）與文件。
- 腳本跑完即可刪除，表的日常運作**全靠內建公式**，不依賴 GAS。

## 資料結構與公式（動欄位前必讀）

- **品名是唯一鍵**：進貨／出貨／總覽全部用品名 `SUMIF` 對回商品主檔，所以主檔品名不可重複；同名不同版（一般版／中文版）要在品名裡寫清楚。
- 進貨紀錄 `E`（小計）＝`ARRAYFORMULA(數量×進貨單價)`；出貨紀錄 `F` 同理。平均進價＝`SUMIF(小計)/總進貨`，靠這兩欄，**不要在小計欄手動打字**（會蓋掉 ARRAYFORMULA 的溢出）。
- 庫存總覽 `A2` 用 `FILTER` 從商品主檔溢出品名＋平台，`C:F` 全是 ARRAYFORMULA；只有 `G`（安全庫存）、`H`（備註）手填。若總覽出現 `#REF!`，多半是有人在 A~F 手動輸入擋住溢出，清掉即可。
- 低庫存標紅＝條件格式自訂公式 `=AND($A2<>"",$G2<>"",$E2<=$G2)`，套在 `A2:H1000`。

## 改版方式

- 改下拉選項（平台／類型／通路）：改 `setup.gs` 開頭的常數陣列後重跑 `setup`；或直接在表上改資料驗證（但下次重跑會被腳本蓋回，建議改腳本為準）。
- 驗證／格式只套到第 1000 列（`MAX_ROWS`），資料超過就調大重跑。
- 重跑 `setup` 安全：只重寫第 1 列標題、格式、驗證與公式儲存格，資料列不動。

## 注意事項

- 成本／進價屬敏感資料，**試算表本身不要開公開連結**；本 repo 只有腳本沒有資料，上傳 GitHub 無虞。
- 刪除商品主檔的列之前，先確認進出貨紀錄是否還需要對帳（刪主檔列後總覽就不再列出該品，但紀錄仍在）。
- `requireValueInRange` 的品名下拉引用主檔 `A2:A1000`，主檔若改名（分頁名），要同步改 `setup.gs` 的 `SHEET_MASTER` 並重跑。
