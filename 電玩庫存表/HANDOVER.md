# 電玩庫存表 — 交接細節

## 架構

- 資料本體＝Google 試算表（網址見 `OVERVIEW.md`）；repo 只有腳本與文件，無實際資料。
- 資料流：Downloads 兩個檔 → `匯入轉換.py` 組 payload → POST GAS Web App（`doPost`）→ 13 分頁整份覆蓋＋2 歷史分頁各寫一欄。
- Web App 驗證：token 存 GAS 指令碼屬性 `API_TOKEN` 與本機 `config.local.json`（gitignore），要一致。
- **改了 setup.gs 要「部署 → 管理部署作業 → 編輯 → 新版本」才生效**（最常忘）。
- 演進：v1 進出貨流水帳（廢棄）→ v2-v4 單檔六/七欄＋可賣量歷史 → v5（2026-07-20）雙檔 13 分頁（git 歷史可查）。

## 輸入檔格式

1. **全站廠商商品匯出 CSV** `20########_XXXX.csv`：UTF-16、tab 分隔、有引號欄，14 欄，約 1.2 萬筆 134 廠商。
   用到欄位索引：0 廠商名稱、1 廠商編號、2 商品編號(20碼)、3 商品名稱、5 可賣量、6 成本、7 售價(網路價)、12 館名稱、13 來源廠商。
   ※ 內含 pchome採購（7777）全部商品，與舊的採購專用 .xls 完全重複（已驗證），故 .xls 停用。
2. **即時業績 xlsx** `MMDD-MMDD即時業績.xlsx`：一週訂單明細 76 欄；檔名就是週區間（轉成 `07/13-07/19` 標籤）。
   用到欄位索引：0 狀態、5 商品館、8 商品ID、9 商品名稱、10 數量、11 退貨數量、12 總業績、13 總成本、28 廠商ID、30 來源廠商ID。
   dimension 標記是壞的，openpyxl 不能用 read_only 模式讀（會只看到 1 格）。

## 商業規則

- **廠商歸戶**：來源廠商(ID) 有值用它、否則用廠商編號/廠商ID。兩檔同規則。
  ⇒ 傑仕登頁＝直屬寄倉＋經 7777 採購的商品；同一商品編號會同時出現在「庫存表」與該廠商頁（重複是刻意的，兩個面向）。
- **退貨**：狀態含「退貨」的明細整筆反向（數量/業績/成本取負）；一般列另扣「退貨數量」欄。此欄實務上多為 0。
- **週銷=0 也列**（銷量頁全品項）；本週有賣但不在匯出檔的商品補在頁尾，館名稱註記「（不在匯出檔）」（多為已下架/組合品）。
- 毛利額＝總業績−總成本；毛利率＝毛利額÷總業績（業績 0 時留白）。可售週數＝可賣量÷週銷（週銷 0 留白）。
- 廠商頁清單改動：改 `匯入轉換.py` 開頭 `STOCK_VENDORS`／`SALES_VENDORS`（可多家合一頁，如英益誠＋艾格瑞）。分頁名用正確字（匯出檔的「宏（其＋石）」＝宏碁、「誠（其＋石）」＝誠碁）。

## payload 格式（POST JSON）

```
{token, date: "2026-07-20"（CSV 檔名日期）, weekLabel: "07/13-07/19"（業績檔名）,
 tabs: [{name, type: "stock"|"sales", headers: [...], rows: [[...]]}],
 qtyHistory: [[商品編號, 商品名稱, 可賣量]],    → 可賣量歷史，欄標籤=date
 salesHistory: [[商品編號, 商品名稱, 週銷]]}    → 週銷歷史，欄標籤=weekLabel
```
- 分頁整份覆蓋（標題列每次重寫，週銷欄帶日期區間）；缺分頁 doPost 自動建＋套格式。
- 歷史分頁：商品編號對列，同標籤重跑覆蓋同一欄；新商品補列；沒出現的留白。
- payload 約 1.4MB；本機備份 `reports/payload_YYYYMMDD.json`（含成本，gitignore）。

## setup.gs 版面

- stock（12 欄，庫存表/傑仕登/宏碁遊戲）：B 文字格式、I 可售週數 0.0、L 毛利率 0.0%。
  條件格式三層（順序即優先序）：深紅 `=AND($B2<>"",$D2=0,$H2>0)`、淺紅 `=AND($B2<>"",$D2=0)`、橘 `=AND($B2<>"",$D2>0,$H2>0,$I2<1)`。
- sales（7 欄）：A 文字格式、G 毛利率 0.0%。
- 格式套用上限：分頁 `TAB_MAX_ROWS`=3000（誠碁 ~2000 筆最大）、歷史 `HIST_MAX_ROWS`=12000（週銷歷史 ~6200 列）。不夠調大後重跑 setup＋新版本部署。
- 只有新建分頁會套格式；要重套既有分頁＝跑 setup（僅庫存表）或先刪分頁讓 doPost 重建。

## 注意事項

- 成本/毛利敏感，**試算表不要開公開連結**；`reports/` 與 `config.local.json` 已 gitignore。
- GAS 單次執行上限 6 分鐘；目前整輪約 1 分內。若未來週銷歷史列數逼近上限、變慢，考慮把歷史改增量。
- `匯入轉換.py` 的 urllib POST 後 GAS 回 302 轉址屬正常，urllib 會跟轉拿 JSON 結果。
