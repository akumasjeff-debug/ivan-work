# 電玩庫存表 — 專案總覽

自家實體電玩商品的**進出貨庫存管理**，本體是一份 **Google 試算表**（取代 Excel 手記）。
本資料夾放的是建置腳本與文件；實際資料都在雲端試算表上。

## 形式

- Google Sheet 四個分頁，公式自動彙算庫存，不需伺服器、手機也能記。
- `apps-script/setup.gs`＝一鍵建置腳本：開新試算表 → Apps Script 貼上 → 執行 `setup` 就長好整份表（步驟見腳本開頭註解）。
- 重跑 `setup` 不會清資料，只重設標題／格式／下拉／公式（欄位改版時用）。

## 試算表結構

| 分頁 | 用途 | 誰填 |
|------|------|------|
| 商品主檔 | 一商品一列：品名（唯一鍵）、平台、類型、條碼、預設成本、售價 | 手填；新商品先建這裡 |
| 進貨紀錄 | 一筆進貨一列：日期、品名（下拉）、數量、進貨單價、廠商；小計自動 | 手填 |
| 出貨紀錄 | 一筆出貨一列：日期、品名（下拉）、數量、通路（PChome/momo/蝦皮/店面/其他）、出貨單價；小計自動 | 手填 |
| 庫存總覽 | 自動：總進貨、總出貨、現有庫存＝進−出、平均進價；低於安全庫存整列標紅 | 只手填「安全庫存、備註」兩欄 |

## 使用流程

1. 新商品 → 先在「商品主檔」加一列。
2. 進貨／出貨時各記一列（品名用下拉選，避免打錯字對不上）。
3. 「庫存總覽」自動更新；想被提醒補貨就填「安全庫存」。
